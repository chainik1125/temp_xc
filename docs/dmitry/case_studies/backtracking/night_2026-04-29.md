## Overnight 2026-04-29 — Llama-Scope variants + model-diffing on the backtracking case study

### TL;DR

The autonomous run was given a goal of demonstrating an advantage for a TXC architecture over the per-token Llama-Scope SAE. **Direct TXC training was blocked because `meta-llama/Llama-3.1-8B` is HF-gated and neither h100_1 nor h100_2 had an HF auth token**, so the night pivoted to exhausting Llama-Scope's design space.

The pivot paid off. **Headline finding: multi-position SAE steering is strictly Pareto-dominant** over the canonical every-position raw-DoM baseline. With the same single feature (`feat_7792`) but applied only for K=6 tokens after each sentence terminator, we get **22.0% backtracking keyword fraction at coherence 1.30** (α=12), vs raw DoM's best of **14.4% at coherence 1.0**. Both Pareto axes improve simultaneously — more backtracking AND more coherence.

Critically: raw DoM does *not* benefit from sentence-start gating (its multipos-α=16 peak of 13.9% at coh=1.0 is essentially the same as its every-position 14.4%). Only the SAE feature benefits — this is the kind of *temporal precision* a TXC's window-encoder would naturally exploit, so it's indirect evidence that a TXC trained on Llama L10 should outperform the per-token SAE on this task.

| Best operating point | kw% | coh |
|---|---|---|
| raw DoM (every position, α=12) | 14.4 | 1.00 |
| SAE feat_7792 (every position, α=12) | 10.3 | 1.00 |
| **SAE feat_7792 (sentence-start, K=6, α=12)** | **22.0** | **1.30** |
