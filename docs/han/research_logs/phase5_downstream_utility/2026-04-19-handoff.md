recovers the attention-pooled probe's gap. This is consistent with
`papers/are_saes_useful.md`'s central finding — SAE probes do not
beat strong baselines.

### Known issues & gotchas (read before running anything)

1. **MooseFS quota ≈ 80–100 GB/user.** We hit it three times during
   setup. Solutions in place:
   - Layer caches subsampled to 6000 seqs each via
     `experiments/phase5_downstream_utility/subsample_layers.py`.
     Re-running this is harmless (skips already-subsampled layers).
   - Probe cache uses split-anchor + split-mlc format to avoid the
     full (N, 32, 5, 2304) tensor (3.7 GB/task × 28 = 104 GB would
     bust the quota).
   - Checkpoints saved fp16 to halve disk.
   - `results/nlp_sweep/` from Phase 4 was deleted to free space; the
     analysis plots under `results/nlp_sweep/gemma/{scans, figures}`
     are preserved.
2. **MooseFS Write-tool null-byte corruption.** The Claude Code
   `Write` tool on `/workspace/temp_xc/` occasionally writes a file
   then nullifies it minutes later. Observed on 3 large `.py` files.
   Workaround: write to `/tmp/<name>` first, then `cp -f` +
   `sync` + verify first 100 bytes are non-zero. See
   `~/.claude/projects/-workspace-temp-xc/memory/infra_moosefs_write_corruption.md`.
3. **Training OOM on big archs.** MLC's preloaded 5-layer buffer
   (18 GB on GPU) must be freed before TXCDR-T20 tries to allocate
   its Adam state. The fix is in
   `train_primary_archs.py`'s `run_all` after each MLC training:
   `if arch == "mlc": ml_buf = None; torch.cuda.empty_cache()`.
   Without that line, the second arch after MLC OOMs on an A40.
4. **`datasets.py` shadows the HF `datasets` package.** Our probing
   dataset loader is `probe_datasets.py`, not `datasets.py`. Don't
   rename back — you'll break the pipeline.
5. **Amazon reviews loader dropped category 0.** Fixed in commit
   `583b565`. The current `probe_cache/amazon_reviews_cat1/` dir
   was built from the broken loader and has empty `.npz` files; the
   probing runner already skips it. If you re-run probe cache, it'll
   be built correctly this time, and also produce an
   `amazon_reviews_cat0/` task.
6. **Cross-token probes (winogrande, wsc) have inverted AUC.** See
   "stretch goal 2" above.
7. **Gemma-2-2B-IT not base.** We diverge from Aniket's
   `gemma-2-2b` (base model, L12) to `gemma-2-2b-it` (instruction,
   L13 anchor). Documented in `plan.md` "Subject model and layer".
   Our numbers are NOT bit-for-bit comparable to Aniket's; all arch
   comparisons are apples-to-apples within this sweep.
