## objective

The Temporal Feature Analysis (TFA) architecture (Lubana et al., "Priors in Time") decomposes sparse autoencoder reconstruction into a **predictable component** (derived from causal attention over previous positions) and a **novel component** (a standard TopK SAE on the residual). The claim is that when features exhibit temporal persistence, TFA can reconstruct persistent features "for free" via the predictable component, leaving the novel component's sparsity budget for genuinely new features. A standard SAE, which processes each token independently, has no such mechanism.

**The goal of this project is to test whether TFA actually delivers on this claim, using synthetic data where the temporal correlations are known by construction.** We want to understand: does TFA provide a measurable reconstruction advantage? Under what conditions? Does the advantage scale? And does TFA's internal decomposition (predictable vs novel) align with the ground-truth temporal structure?

## suggested reading

Before diving in, the following materials provide useful context. Read at least the TFA paper; the others are helpful but not essential.

**Primary:**

- [[priors_in_time]] (Lubana et al.) — The paper that introduces TFA. Pay particular attention to:
  - **Section 4** (Temporal Priors of Sparse Autoencoders): Proposition 4.2 argues that a restrictive sparsity budget forces standard SAEs to "switch" feature support across positions, losing persistent features. This is the theoretical motivation for TFA.
  - **Section 5** (Temporal Feature Analysis): The architecture description, especially how the predictable component uses causal attention and how the novel component operates on the residual.
  - **Appendix B.1** (Training procedure and activations normalization): Describes TFA's input scaling convention and training setup.

**Secondary:**

- [[sparse_but_wrong]] — Shows that setting SAE L0 incorrectly causes feature mixing: too low and the SAE merges correlated features, too high and it finds degenerate solutions. Directly relevant to our sparsity sweep — we should be aware that the baseline SAE's behavior at different $k$ values is not just "less reconstruction" but qualitatively different.
- [[temporal_sae]] — A *different* temporal SAE approach (not TFA). Uses a contrastive loss to encourage consistent activations over adjacent tokens, partitioning features into "semantic" (slow) and "syntactic" (fast). Worth knowing about as an alternative architecture, though we are evaluating TFA specifically.
- [[autoencoding_slow]] — Background on the slowness principle: the idea that underlying generative factors change more slowly than raw observations. TFA's predictable component is essentially a learned slowness prior. This paper formalizes the idea in the VAE setting.

**Also useful:**

- [[toy_model_idea]] — Describes Scheme C (the Markov chain data generation we use) and the rationale behind it.

## data generation

We use Scheme C from [[toy_model_idea]], which gives us full control over temporal correlations. The generative process is described below; the specific parameter choices ($n$, $\pi_i$, $\rho_i$, $d_\text{hidden}$) are up to whoever runs the experiments.

**Generative process.** Let $\mathbf{f}_1, \ldots, \mathbf{f}_n \in \mathbb{R}^d$ be $n$ orthogonal unit-norm feature directions. At each sequence position $t$, the activation vector is

$$\mathbf{x}_t = \sum_{i=1}^{n} s_{i,t} \, \mathbf{f}_i$$

where $s_{i,t} \in \{0, 1\}$ is the support indicator for feature $i$ at position $t$. We use unit magnitudes ($m_{i,t} = 1$ when active) to isolate the effect of temporal support correlations.

**Support process.** For each feature $i$ independently, the support sequence $(s_{i,1}, s_{i,2}, \ldots)$ follows a two-state Markov chain with transition probabilities

$$p_{01}^{(i)} = \pi_i(1 - \rho_i), \qquad p_{10}^{(i)} = (1 - \pi_i)(1 - \rho_i)$$

where $\pi_i$ is the marginal activation probability (stationary distribution) and $\rho_i = 1 - p_{01}^{(i)} - p_{10}^{(i)}$ is the lag-1 autocorrelation. The lag-$k$ autocorrelation decays geometrically: $\text{Corr}(s_{i,t}, s_{i,t+k}) = \rho_i^k$.

**Feature independence.** Features are independent of one another: $s_{i,t}$ and $s_{j,t'}$ are independent for $i \neq j$. Temporal correlations exist *within* each feature across positions, but there are no cross-feature correlations.

**Expected L0.** Since features are independent with marginal probability $\pi_i$, the expected number of active features per position is $\mathbb{E}[L_0] = \sum_{i=1}^{n} \pi_i$.

**Example configuration.** One possible starting point is $n = 10$ features with $\pi_i = 0.2$ for all $i$ (giving $\mathbb{E}[L_0] = 2.0$) and a spread of autocorrelation strengths across features (e.g. some i.i.d. features at $\rho = 0$, some weakly persistent at $\rho = 0.3$, some strongly persistent at $\rho = 0.9$). The key requirement is that the data includes a range of temporal persistence strengths so we can see whether TFA's behavior varies with $\rho$. The specific distribution, number of features, and hidden dimension are design choices — feel free to change them if a different setup is more informative.

## TFA architecture

TFA (`TemporalSAE` from the "Priors in Time" codebase, in `src/v2_temporal_schemeC/tfa/`) has two components:

- *Predictable component:* Encodes activations into a latent space, applies causal multi-head attention (each position attends only to previous positions), then decodes back to input space. This captures features that are predictable from prior context.
- *Novel component:* A TopK SAE applied to the residual after subtracting the predictable component's reconstruction. This captures features that are new at the current position.

The full reconstruction is $\hat{\mathbf{x}}_t = \hat{\mathbf{x}}_t^\text{pred} + \hat{\mathbf{x}}_t^\text{novel}$.

**Key parts of the source code** (in `tfa/saeTemporal.py`) worth reading carefully:

- **Line 36** (`self.lam = 1 / (4 * dimin)`): The fixed scaling constant $\lambda$. This is *not learned* — it is hardcoded to assume $\mathbb{E}[\|\mathbf{x}\|] = \sqrt{d_\text{in}}$. If the input data has a different mean norm, the encoder operates in a bad regime. This has practical consequences for any synthetic data setup (see "things to watch out for" below).
- **Lines 80--81** (encoding and causal shift): The encoder computes `z_input = ReLU(x * lam @ E)`, then constructs `z_ctx` by shifting `z_input` forward by one position (prepending zeros). This is how causality is enforced — position $t$ attends only to codes from positions $< t$.
- **Lines 84--98** (projective attention): The attention output is decoded back to input space, then *projected* onto the current input residual. This projection step is subtle — it scales the predicted reconstruction to match the input rather than taking it at face value.
- **Lines 104--114** (novel component): A standard TopK SAE applied to whatever the predictable component could not explain.
- **Line 151** (final reconstruction): `x_recons = (z_novel + z_pred) @ D + b`.

The `tfa/utils.py` file contains the `ManualAttention` implementation — a standard causal multi-head attention with custom weight initialization.

**Input scaling caveat.** Because of the hardcoded $\lambda$, the synthetic data's mean norm matters. If $\mathbb{E}[\|\mathbf{x}\|] \neq \sqrt{d_\text{in}}$, one approach is to scale all inputs (for both TFA and whatever baseline is used) by $\sqrt{d_\text{in}} / \mathbb{E}[\|\mathbf{x}\|]$ so both models see identical data at the scale TFA expects. There may be other ways to handle this — the important thing is that any scaling is applied symmetrically and doesn't give either model an unfair advantage.

## what we want to find out

The core experiment is a comparison of TFA against a baseline that does not have access to temporal structure. The main questions:

1. **Does TFA reconstruct better than the baseline?** Specifically, does it achieve lower reconstruction error at the same sparsity level?
2. **Under what conditions does TFA help?** We expect TFA's advantage to be largest when the sparsity budget $k$ is binding ($k < L_0$) and to vanish when the baseline has enough capacity ($k \geq L_0$). Is this true?
3. **Does TFA's internal decomposition make sense?** Does the predictable component preferentially capture high-$\rho$ features and leave low-$\rho$ features to the novel component?
4. **Does the advantage scale?** At larger $L_0$ (more features), the gap between $k$ and $L_0$ widens — does TFA's advantage grow accordingly?

An ideal deliverable would be a **sparsity vs reconstruction loss Pareto frontier** where each point is a trained model. If TFA works as claimed, its curve should lie below the baseline's curve in the low-sparsity regime.

### baseline

The natural baseline is a standard SAE that processes each token independently — it takes a single vector $\mathbf{x}_t \in \mathbb{R}^d$ and produces $\hat{\mathbf{x}}_t$ with no access to neighboring positions, positional information, or sequence structure. During training, tokens from all sequence positions are pooled into a flat dataset; minibatches are drawn from this pool. Since the SAE has no within-batch interactions (each token is encoded and decoded independently), the ordering of tokens within a minibatch does not affect training.

A `BatchTopKTrainingSAE` from sae-lens is one option. But other baselines might be more informative — for example, a position-aware SAE, or any other architecture that lacks causal attention but might otherwise be competitive. The choice of baseline is up to whoever runs the experiments; the important thing is that it does not have access to temporal context.

### parameters to sweep

To trace out the Pareto frontier, some parameters worth considering:

- **$k$ (TopK sparsity):** The primary knob that directly controls the sparsity--reconstruction tradeoff.
- **Dictionary width (expansion factor):** A wider dictionary gives the encoder more directions to choose from at the same $k$. In practice, SAEs at LLM scale use expansion factors of 4--32$\times$.
- **TFA architecture capacity:** Number of attention layers, number of heads, bottleneck factor. More capacity may improve the predictable component but adds parameters.

These are suggestions — there may be other axes worth exploring.

### metrics to consider

These are starting points; feel free to use different metrics if they are more informative.

- **Reconstruction loss.** MSE averaged over tokens is the obvious choice. Since both models should see identically scaled data, MSE is directly comparable.
- **Sparsity axis.** The TopK hyperparameter $k$ is one option for the Pareto x-axis, since it is the knob we control for both models. An alternative is the *measured* L0 (actual number of nonzero codes per token). For TFA, the novel component's L0 $= k$, but the predictable component contributes additional reconstruction not captured by L0. Which x-axis is more informative may depend on the results.
- **TFA diagnostics.** The predictable energy fraction (how much of TFA's reconstruction comes from temporal prediction) and per-feature predictable fraction (whether high-$\rho$ features are routed to prediction) are useful for understanding TFA's internal behavior, but are not evaluation metrics.

### things to watch out for

- **TFA convergence.** TFA's attention mechanism is more complex than a standard SAE encoder and may need substantially more training steps. Evaluating too early could make TFA look worse than it is. It is worth checking convergence explicitly before drawing conclusions.
- **Input scaling.** See the caveat in the TFA architecture section. Whatever approach is taken, both models should see comparable data.
- **Single-seed noise.** Small differences (especially near $k = L_0$) may not be statistically significant. Multiple seeds are worth running for any close comparisons.
