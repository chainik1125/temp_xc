# TinyStories sleeper: TXC vs MLC baseline

Compares single-feature prompt-only perturbation ablation between a
cross-layer crosscoder (MLC, L=5 residual points) and a temporal crosscoder
(TXC, T=30 token window) at three layers of the public sleeper-agent
TinyStories-Instruct-33M model.

Full spec: `../../notes/tinystories_sleeper_crosslayer_vs_crosstime.md`.
Implementation plan: `~/.claude/plans/cryptic-watching-sifakis.md`.

## Run

```bash
# On a40_climb (CUDA)
python harvest_activations.py
python train_crosscoders.py
python run_ablation_sweep.py
python plot_pareto.py
```

Each step caches to `outputs/data/*.pt`; downstream steps read those caches.
Defaults match the plan (d_sae=1536, k_total=32, T=30, 8000 steps, δ=0.05 nats).

## Outputs

- `outputs/data/tokens_cache.pt` — paired tokens + is_deployment + story_marker_pos
- `outputs/data/activations_cache.pt` — (N, 128, 5, 768) fp16 residual acts
- `outputs/data/crosscoder_{mlc,txc_early,txc_mid,txc_late}.pt`
- `outputs/data/feature_rankings_{arch}.json/.pt`
- `outputs/data/val_sweep_{arch}.json` — candidate (f, α) metrics
- `outputs/data/test_results.json` — held-out test metrics
- `outputs/plots/pareto_asr_vs_utility.png`
- `RESULTS.md` — final comparison table (generated by `plot_pareto.py`)
